# cns-cn-pages

本仓库用于归档和发布多个独立 HTML 页面。这些页面内容相互独立，不属于任何单一网站或域名，仅作为静态 HTML 文件集合进行管理和分发。

## 项目简介

cns-cn-pages 是一个面向一般归档用途的静态页面仓库。仓库中的 HTML 文件均保持原始结构，不依赖特定后端服务或动态数据源，可直接通过 GitHub Pages 或其他静态托管服务访问。

主要用途包括：
- 归档历史版本页面
- 存放独立的说明页、索引页或导航页
- 作为临时或长期页面发布的中转仓库

## 目录说明

```
cns-cn-pages/
├── index.html          # 仓库首页（可选）
├── pages/              # 页面归档目录
│   ├── example1.html
│   ├── example2.html
│   └── ...
├── assets/             # 静态资源（CSS、JS、图片等）
│   ├── css/
│   ├── js/
│   └── images/
└── README.md           # 本文件
```

- `pages/`：存放所有独立 HTML 文件，按需组织子目录。
- `assets/`：存放页面引用的公共资源文件，建议按类型分类存放。
- `index.html`：仓库根目录下的默认首页，可自定义或留空。

## 页面归档说明

- 所有页面均为纯静态 HTML，不包含服务器端脚本。
- 页面间无强制依赖关系，每个文件可独立访问。
- 资源引用建议使用相对路径，以确保在不同部署环境下正常工作。
- 归档页面可能包含外部链接，请自行评估其可用性和安全性。

## 维护说明

本仓库由维护者不定期更新。提交内容遵循以下原则：

- 仅添加或更新 HTML 文件及其相关静态资源。
- 不修改其他用户或贡献者已归档页面的原始内容，除非必要修复。
- 保持目录结构清晰，避免文件冗余堆积。
- 如需贡献新页面，请提交 Pull Request，并说明页面用途。
